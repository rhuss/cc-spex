# Implementation Plan: Harden spex-detach

**Branch**: `042-detach-hardening` | **Date**: 2026-07-14 | **Spec**: [spec.md](spec.md)

## Summary

The spex-detach extension exists but has three gaps: the finish command has no detach integration, the brainstorm skill references a wrong script path, and there is no post-detach verification. This plan fixes all three gaps and adds archive move semantics, brainstorm archiving, .gitignore warning, and sibling repo discovery.

## Implementation Approach

### Phase 1: Fix bugs and add verification (P1 + P2 requirements)

**Task 1: Fix brainstorm script path bug (FR-005)**

In `speckit.spex.brainstorm.md` line 282, change:
```
DETACH_SCRIPT=".specify/extensions/spex/scripts/spex-detach.sh"
```
to:
```
DETACH_SCRIPT=".specify/extensions/spex-detach/scripts/spex-detach.sh"
```

**Task 2: Add `verify` subcommand to spex-detach.py (FR-006, FR-007)**

Add `cmd_verify(args)` to `spex-detach.py`:
- Accept `--branch` and `--base` arguments
- Run `git diff --name-only <base>..<branch>`
- Check each file path against SpecKit fingerprint patterns derived from `strip_paths` config
- Output JSON: `{"clean": true/false, "leaked_files": [...], "patterns_checked": [...]}`
- Exit 0 if clean, exit 1 if leaked files found

Integrate verification into `cmd_detach`: after creating the PR branch and committing, call the verify logic on the new branch. If verification fails, delete the PR branch and report the error.

**Task 3: Add .gitignore check to spex-detach.py (FR-008)**

Add `cmd_check_gitignore(args)` or integrate into `cmd_detach`:
- Check if `upstream` remote exists: `git remote | grep -x upstream`
- If yes, read `.gitignore` and check for `.specify`, `specs`, `brainstorm`
- Emit warning to stderr for any missing entries
- Non-blocking (does not affect exit code)

**Task 4: Add move semantics and brainstorm archiving to archive (FR-004, FR-012)**

Modify `cmd_archive` in `spex-detach.py`:
- Add `--move` flag: after successful copy+commit, delete source directories
- Add brainstorm archiving: copy `brainstorm/` to sibling repo alongside `specs/<feature>/` and `.specify/`
- When `--move` is set, delete `specs/<feature>/` and `brainstorm/` from source (defer `.specify/` deletion to avoid breaking hooks)
- Handle deletion failures gracefully (warn, don't fail)

### Phase 2: Wire detach into finish command (P1 requirements)

**Task 5: Add detach step to Phase 3 of speckit.spex.finish.md (FR-001, FR-002, FR-003, FR-011)**

Insert a detach step between Phase 3 Step 1 (Commit outstanding changes) and Phase 3 Step 2 (Compute merge base). Per FR-001, detach must run after committing but before the squash. Per FR-004, source directory deletions from `archive --move` must be included in the squash commit, which requires the archive to have already staged those deletions.

**Phase 3, Step 1.5: Detach (when spex-detach is enabled)**

Detection:
```bash
DETACH_ENABLED=false
if [ -d ".specify/extensions/spex-detach" ]; then
  DETACH_ENABLED=true
fi
```

If enabled:
1. Check for `--skip-archive` flag (parsed in Argument Parsing section)
2. If archive not skipped and `archive.path` is configured:
   - Run `spex-detach.sh archive --move --include-brainstorm`
   - Report archive result
3. If archive not skipped but `archive.path` not configured:
   - Emit warning: "archive.path not configured, skipping archive"
4. Run `.gitignore` check (non-blocking warning)
5. Run `spex-detach.sh detach` to create clean `pr/<branch>` branch
6. Verification runs automatically as part of detach (integrated in Task 2)
7. Set `DETACH_PR_BRANCH` variable with the clean branch name

Update Phase 4 (Select Action):
- When `DETACH_PR_BRANCH` is set, add "Push clean PR branch" option
- This option runs: `git push <remote> <pr-branch>`

Update Argument Parsing:
- Add `--skip-archive` flag alongside existing `--no-smoke-test`

### Phase 3: Brainstorm enhancements (P3 requirements)

**Task 6: Add sibling repo brainstorm discovery (FR-009)**

In `speckit.spex.brainstorm.md`, in the revisit detection section (step 4 of checklist):
- Check if spex-detach is enabled and `archive.path` is configured
- If yes, also scan `<archive.path>/brainstorm/` for `NN-*.md` files
- Include sibling matches in the revisit detection results alongside local matches
- If user chooses to update a sibling brainstorm, write the revisit section to the sibling file

**Task 7: Add init auto-detection hint (FR-010, deviation noted)**

FR-010 specifies `specify init` as the integration point, but `init` is upstream spec-kit and cannot be modified here (see "No modify speckit files" policy). As a practical alternative, the auto-detection advisory is placed in the brainstorm skill, which is the earliest touchpoint where a missing `archive.path` surfaces.

In `speckit.spex.brainstorm.md`, when the detach check runs and `archive.path` is empty:
- Scan `../` for directories matching `*-specs`
- If found, suggest it: "Detected sibling specs repo at ../<name>. Set archive.path in spex-detach-config.yml to use it."
- This is advisory only, does not modify config
