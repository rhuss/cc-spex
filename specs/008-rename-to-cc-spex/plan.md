# Implementation Plan: Rename Plugin to cc-spex

**Branch**: `008-rename-to-cc-spex` | **Date**: 2026-03-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-rename-to-cc-spex/spec.md`

## Summary

Rename the SDD plugin from `cc-sdd` to `cc-spex`, changing the command prefix from `sdd:` to `spex:`, the plugin root directory from `sdd/` to `spex/`, and all internal references. Includes backwards-compatible migration for existing projects using `sdd-traits.json`. The methodology name "SDD" stays in prose; only tooling identifiers change.

## Technical Context

**Language/Version**: Bash (POSIX-compatible), Python 3 (hooks), Markdown (commands/skills)
**Primary Dependencies**: `jq` (JSON parsing), `specify` CLI (spec-kit), Claude Code plugin system
**Storage**: File-based (JSON config, Markdown documents)
**Testing**: Manual verification via `make reinstall` + Claude Code session testing
**Target Platform**: macOS/Linux (Claude Code CLI environments)
**Project Type**: Claude Code plugin (Markdown/Bash, no compiled artifacts)
**Performance Goals**: N/A (rename operation, not runtime feature)
**Constraints**: Backwards compatibility with existing `sdd-traits.json` configs
**Scale/Scope**: ~70 files to update, 28 distinct pattern types, 60+ replacement locations

## Implementation Phases

### Phase 1: Plugin Internal Rename (P1 - Core)

The bulk mechanical rename inside the plugin directory. Must be done atomically so the plugin remains internally consistent.

**Step 1.1: Rename plugin root directory**
- `git mv sdd/ spex/`
- This is the foundational operation; all subsequent changes happen in `spex/`

**Step 1.2: Rename shell scripts**
- `git mv spex/scripts/sdd-init.sh spex/scripts/spex-init.sh`
- `git mv spex/scripts/sdd-traits.sh spex/scripts/spex-traits.sh`

**Step 1.3: Update command files (10 files)**
- In each `spex/commands/*.md`: change frontmatter `name: sdd:*` to `name: spex:*`
- Update any `{Skill: sdd:*}` references to `{Skill: spex:*}`
- Update help text referencing `/sdd:*` commands

**Step 1.4: Update skill files (14 files with references)**
- In each `spex/skills/*/SKILL.md`: replace `{Skill: sdd:*}` with `{Skill: spex:*}`
- Replace `/sdd:` command references with `/spex:`
- Replace `sdd-init.sh` / `sdd-traits.sh` path references
- Replace `sdd-traits.json` config references

**Step 1.5: Update overlay files (9 files)**
- Replace `{Skill: sdd:*}` with `{Skill: spex:*}`
- Replace `/sdd:` command references with `/spex:`
- Keep `<!-- SDD-TRAIT:name -->` sentinel markers unchanged (methodology name)

**Step 1.6: Update hook scripts (4 Python files)**
- context-hook.py: `/sdd:` prefix matching to `/spex:`, XML tags `<sdd-*>` to `<spex-*>`, marker files `.claude-sdd-*` to `.claude-spex-*`, script paths, config paths
- skill-gate-hook.py: marker file pattern `.claude-sdd-*` to `.claude-spex-*`, `/sdd:` references to `/spex:`
- verification-gate-hook.py: marker pattern, `/sdd:verify` reference
- teams-enforce-hook.py: `{Skill: sdd:*}` reference

**Step 1.7: Update shell scripts (internal references)**
- spex-init.sh: `sdd-traits.sh` references to `spex-traits.sh`, `sdd-traits.json` to `spex-traits.json`
- spex-traits.sh: `TRAITS_CONFIG` path, `SDD_PATTERN_INIT/TRAITS` permission patterns, script self-references, config file references
- check-upstream-changes.sh: any `sdd` path references

**Step 1.8: Update plugin metadata**
- `spex/.claude-plugin/plugin.json`: name `"sdd"` to `"spex"`, keyword `"sdd"` to `"spex"`
- `spex/.claude/commands/update-superpowers.md`: `cc-sdd` references to `cc-spex`
- `spex/.superpowers-sync`: update metadata references

**Step 1.9: Update docs (6 files)**
- All `sdd:` command references to `spex:`
- All `sdd-init.sh` / `sdd-traits.sh` references
- All `sdd/` path references

### Phase 2: Backwards-Compatible Migration (P2)

**Step 2.1: Add migration logic to spex-init.sh**
- At the start of `do_init()`, check for `.specify/sdd-traits.json`
- If found and `.specify/spex-traits.json` does not exist: copy `sdd-traits.json` to `spex-traits.json`, print migration message
- If both exist: prefer `spex-traits.json`, print info message
- If only `spex-traits.json` exists: no action

**Step 2.2: Update spex-traits.sh for migration awareness**
- `TRAITS_CONFIG` should check for `spex-traits.json` first, fall back to `sdd-traits.json` if it exists (read-only fallback)
- New writes always go to `spex-traits.json`

**Step 2.3: Update phase marker**
- `.specify/.sdd-phase` references become `.specify/.spex-phase`
- Recognize old `.sdd-phase` during transition
