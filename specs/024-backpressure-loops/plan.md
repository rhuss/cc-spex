# Implementation Plan: Backpressure Loops

**Branch**: `024-backpressure-loops` | **Date**: 2026-06-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/024-backpressure-loops/spec.md`

## Summary

Add two automated feedback mechanisms to spex: (1) per-task test checkpoints during implementation that catch regressions before they compound across tasks, and (2) a post-PR watch mode in the finish command that monitors CI status, auto-fixes failures, and optionally integrates with collab triage for review comment processing.

## Implementation Phases

### Phase 1: Per-Task Test Checkpoints (User Story 1)

Modify the ship pipeline's implement subagent prompt in `speckit.spex.ship.md` to include per-task test checkpoint instructions. The subagent prompt tells the implement command to run the test suite after each task and fix failures before proceeding.

**Files modified**:
- `spex/extensions/spex/commands/speckit.spex.ship.md` (Stage 6 subagent prompt)

**Key decisions from research**:
- Test checkpoints are added via the ship subagent prompt, not by modifying the upstream speckit-implement skill (R1)
- Test command auto-detection reuses verify's existing logic (R3)
- Config opt-out via `implement.test_between_tasks` in spex-config.yml (data model)

### Phase 2: Watch Mode State Management (User Story 4)

Extend the state management script and statusline to support watch mode. This is infrastructure for Phase 3.

**Files modified**:
- `spex/scripts/spex-ship-state.sh` (add `watch-start`, `watch-update`, `watch-cleanup` commands)
- `spex/scripts/spex-ship-statusline.sh` (add watch mode rendering: PR number, elapsed time, CI status)

**Key decisions from research**:
- Single state file extended with watch fields, not a separate file (R4)
- `mode: "watch"` distinguishes from `"ship"` and `"flow"` modes

### Phase 3: Post-PR Watch Mode (User Stories 2, 3)

Add `--watch` flag to the finish command. After PR creation/push, enter a monitoring loop that polls CI status and optionally invokes collab triage.

**Files modified**:
- `spex/extensions/spex/commands/speckit.spex.finish.md` (add --watch flag, watch loop logic)
- `spex/extensions/spex/commands/speckit.spex.ship.md` (pass --watch through to finish stage when --create-pr is set)

**Key decisions from research**:
- Uses ScheduleWakeup for polling persistence within a session (R2)
- Invokes collab triage command directly when collab enabled, no extraction needed (R5)
- New comments detected via `gh api` before invoking triage (R5)

### Phase 4: Documentation and Config

Update documentation to cover the new capabilities.

**Files modified**:
- `README.md` (commands reference, workflow description)
- `spex/docs/help.md` (quick reference)
