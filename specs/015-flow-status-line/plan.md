# Implementation Plan: Flow Status Line

**Branch**: `015-flow-status-line` | **Date**: 2026-04-07 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/015-flow-status-line/spec.md`

## Summary

Extend the spex status line (currently ship-only) to support a flow mode for manual workflow tracking. The status line shows milestone completion (specify, plan, tasks, implement) via artifact detection, review checklist state (spec, plan, code reviews), active traits, and the next recommended step. Rename the state file from `.spex-ship-phase` to `.spex-state` and split the monolithic REVIEWERS.md into per-review artifacts. Add a completion celebration on successful stamp.

## Implementation Phases

### Phase 1: State File Rename (Foundation)

Rename all references from `.spex-ship-phase` to `.spex-state` across the entire codebase. This is a prerequisite for all other phases since every subsequent change touches the state file.

**Files**:
- `spex/scripts/spex-ship-statusline.sh`
- `spex/scripts/spex-ship-state.sh`
- `spex/scripts/hooks/pretool-gate.py`
- `spex/skills/ship/SKILL.md`
- All `.claude/skills/speckit-*/SKILL.md` guard sections
- All overlay files referencing the state file path

**Validation**: `rg '\.spex-ship-phase' spex/ .claude/` returns zero matches after completion.

### Phase 2: Review Artifact Split (P1 - User Story 3)

Update review skills to produce individual review files instead of monolithic REVIEWERS.md.

**Changes**:
- `spex/skills/review-spec/SKILL.md`: Output to `REVIEW-SPEC.md` in spec directory
- `spex/skills/review-plan/SKILL.md`: Output to `REVIEW-PLAN.md` instead of `REVIEWERS.md`
- `spex/skills/review-code/SKILL.md`: Output to `REVIEW-CODE.md` instead of appending to REVIEWERS.md
- `spex/scripts/spex-ship-state.sh`: Update `verify_stage_artifacts()` to check for split files with REVIEWERS.md fallback (FR-015a)
- `spex/skills/ship/SKILL.md`: Update stage artifact references

**Validation**: Review skills produce correct individual files. Ship pipeline accepts both old and new formats.

### Phase 3: Flow Mode Status Line (P1 - User Stories 1, 2)

Add flow mode rendering to the status line script.

**Changes to `spex/scripts/spex-ship-statusline.sh`**:
- Read `mode` field from state file
- If `mode=ship`: existing behavior (unchanged)
- If `mode=flow`:
  - Read `spec_dir` from state file
  - Check artifact existence: `spec.md`, `plan.md`, `tasks.md` in spec_dir
  - Check `implemented` field in state file
  - Check review artifacts: `REVIEW-SPEC.md`, `REVIEW-PLAN.md`, `REVIEW-CODE.md` in spec_dir
  - Compute next step per FR-004 priority
  - Render checkmark line with ANSI colors (FR-006a)

**Display format**:
```
spex ✓spec ✓plan ○tasks ○impl | reviews: ✓S ○P ○C | next: tasks
```
- Green (matching ship scheme) for ✓ completed items
- Dim/gray for ○ pending items
- Highlighted color for next step

**Validation**: Status line renders correctly for various artifact combinations. Silent when no state file exists.

### Phase 4: Trait Display (P2 - User Story 4)

Add trait names to status line output in both modes.

**Changes to `spex/scripts/spex-ship-statusline.sh`**:
- Read `.specify/spex-traits.json` with `jq`
- Extract trait names where value is `true`
- If any traits enabled, append `[trait1, trait2]` in muted color
- If no traits or file missing, omit section

**Display format** (appended to both ship and flow lines):
```
spex ✓spec ✓plan ○tasks ○impl | reviews: ✓S ○P ○C | next: tasks [superpowers, deep-review]
spex:ship 🗺️ plan ▓▓░░░░░░░ 3/9 🧠 [superpowers, worktrees]
```

**Validation**: Traits display correctly. Omitted when none enabled. Updates when config changes.

### Phase 5: Flow Lifecycle Integration (P2 - User Stories 5, 6)

Wire flow state creation and cleanup into the workflow skills.

**Changes**:
- `speckit-specify/SKILL.md`: After spec creation, create `.specify/.spex-state` with `"mode": "flow"` fields (FR-010, FR-016). Only when no state file exists or existing flow state should be overwritten.
- `spex/skills/ship/SKILL.md`: Create state with `"mode": "ship"` (FR-011). Already does this, just needs `mode` field added.
- `speckit-implement/SKILL.md`: On successful completion, update state file to set `"implemented": true` (FR-018).

**Validation**: State file created on specify, mode field correct. Implementation flag set. Ship mode overwrites flow state.

### Phase 6: Celebration (P3 - User Story 7)

Add completion celebration to the stamp/verification skill.

**Changes to `spex/skills/verification-before-completion/SKILL.md`**:
- After all verification passes, check if state file exists
- If state file present:
  - Display ASCII art celebration banner
  - Compute stats: feature name (from `feature_branch`), duration (from `started_at`), review count (count REVIEW-*.md files), commit count (`git rev-list --count main..HEAD`)
  - Display stats summary
  - Select and display random sign-off from pool of 5+ messages
  - Remove state file

**Sign-off pool examples**:
- "Ship it!"
- "Another one bites the dust."
- "That's a wrap."
- "Clean as a whistle."
- "Nailed it."
- "Spec met. Code shipped. Coffee earned."

**Validation**: Celebration displays once on successful stamp. State file removed after. No celebration if state file absent.
