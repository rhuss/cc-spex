## Notable Upstream Changes (Non-Synced)

### Subagent-Driven Development (SDD) Overhaul
530 insertions, 157 deletions across 8 files. Major structural changes:

1. **Reviewer consolidation**: `code-quality-reviewer-prompt.md` and `spec-reviewer-prompt.md` deleted, replaced by unified `task-reviewer-prompt.md` that produces both spec compliance and code quality verdicts in one pass (~15% savings).

2. **Pre-built review packages**: New `scripts/review-package` shell script pre-generates a diff file (commit list + stat summary + full diff with U10 context) that the reviewer reads as a file instead of running git commands (~10% savings).

3. **Task brief script**: New `scripts/task-brief` extracts individual task content from the plan for isolated implementer/reviewer context.

4. **SDD workspace helper**: New `scripts/sdd-workspace` creates `.superpowers/sdd/` directory for artifacts (moved out of `.git/` protected path in 6.0.3).

5. **Terse reviewer contract**: Autoresearch experiment E27 reduced reviewer output by 41% with verdicts intact.

6. **Conditional implementer tiering**: Lighter models (Haiku) used for tasks where the plan carries complete code.

7. **Global Constraints in reviewer prompt**: Spec's project-wide requirements copied verbatim into reviewer prompt as attention lens.

8. **Autoresearch finding (risk)**: Reviewers given only diff packages without the spec "produce confident spec verdicts that silently redefine 'spec' as the global constraints." Validates spex-deep-review's approach of injecting spec text into every agent prompt.
